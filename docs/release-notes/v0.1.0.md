# AlpenLedger 0.1.0 (Build 1)

Status: Draft
Release date: TBD

## Audience

This release is for local pilot validation of AlpenLedger as a macOS-first,
local-first Swiss finance manager. It is not evidence that a production-signed
and notarized artifact has shipped.

## Highlights

- Local encrypted workspace creation and reopening.
- Bank statement and document import flows with immutable raw-source storage.
- Evidence, issue, and proposal workflows for review-first financial work.
- Deterministic personal-tax, VAT, and readiness previews backed by local
  fixtures and rule-pack validation.
- Copilot answer cards with source references and task creation for review.
- Demo workspace, first-run checklist, and native help guide for local pilot
  onboarding.
- Support runbook for private intake, sanitized diagnostics, backup safety, and
  issue hand-off.
- English-first localization framework with German/French language-claim
  boundaries kept planned until translation and layout evidence exist.

## Local-First And Privacy Notes

- Authoritative workspace, document, ledger, tax, and audit data remain local by
  default.
- Default runtime mode rejects unsupported cloud privacy modes and stays in
  local-only mode.
- Support diagnostics and support bundles are sanitized before export.

## Data Integrity And Safety Notes

- Raw imports are stored immutably and can be re-read after source files change.
- Confirmed-write agent tools require explicit input-bound approval.
- Copilot task creation opens review issues and does not mutate ledger or tax
  facts.
- Locked tax years and VAT periods reject changes that would alter closed
  accounting/tax state.

## Known Limitations

- Full UI automation and manual smoke evidence must be captured on a machine
  with macOS automation permission before release.
- Strict release preflight requires real Apple Developer ID and notary
  credentials.
- The packaging command requires a prepared app bundle and only publishes
  release evidence when final artifact verification passes against the staged
  ZIP and checksum sidecar.
- The final signed, notarized, stapled ZIP has not yet been archived as release
  evidence.
- Canton, business-tax, and document-standard coverage remains intentionally
  narrow for the first pilot slice.
- The pilot UI and help copy are English-first and do not yet include German or
  French localization.

## Upgrade And Compatibility Notes

- This is the first release-note draft for the pilot line.
- Workspace migration coverage exists for current schema paths, but release
  candidates still need clean-machine and restored-workspace verification.

## Support Notes

- Use sanitized diagnostics and support bundles for issue triage.
- Follow `docs/support.md` for private support intake, severity, backup safety,
  privacy boundaries, and escalation.
- Do not attach raw workspace databases, source documents, or personal finance
  exports to public issue trackers.
